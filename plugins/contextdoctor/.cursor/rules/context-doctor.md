# Context Doctor 项目规则

## 关于 Context Doctor

Context Doctor 是一个上下文污染检测工具，用于识别和修复 AI 对话中的上下文问题。

## 可用指令

### /contextdoctor
检查当前项目的上下文健康状况，生成 HTML 报告。

**适用场景**：
- 长时间对话后
- 发现 AI 行为异常
- 需要优化对话质量

### /repair
检查污染并提供具体修复方案。

**适用场景**：
- 已发现上下文问题
- 需要立即修复
- 需要清理建议

## 污染类型说明

1. **Skill/Plugin 冲突**
   - 多个工具功能重叠
   - 权限设置冲突
   - 命名空间污染

2. **指令矛盾**
   - 目标变更未明确
   - 约束条件冲突
   - 隐含假设不一致

3. **错误累积**
   - 早期错误影响后续
   - 误解持续传播
   - 信息丢失

## 最佳实践

- 定期运行 `/contextdoctor` 检查健康度
- 在重要决策前检查上下文
- 发现问题及时使用 `/repair`
