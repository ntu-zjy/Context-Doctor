## 核心组件

### 1. 检测器 (Detector)

负责从对话历史中提取信息，识别潜在的污染来源。

**功能**：
- 提取已加载的 Skills/Plugins 列表
- 扫描用户指令历史
- 追踪错误传播路径

### 2. 分析器 (Analyzer)

对检测到的数据进行深度分析，识别具体的污染类型。

**功能模块**：
- **Skill Checker**: 检查技能冲突和冗余
- **Conflict Detector**: 检测指令矛盾
- **Error Tracker**: 追踪错误累积

### 3. 评分引擎 (Score Engine)

计算综合健康评分，基于污染的类型、数量和严重程度。

**评分算法**：
```
基础分 = 100
扣分 = Σ(问题权重 × 问题数量)
最终分 = max(0, 基础分 - 扣分)

权重：
- Critical: 30
- Warning: 15
- Suggestion: 5
```

### 4. 报告器 (Reporter)

生成可视化的 HTML 报告。

**报告结构**：
- 头部：标题和综合评分
- 概览：统计数据和分布图表
- 详情：问题列表和引用
- 建议：修复方案（repair 模式）

## 污染类型

### Type 1: Skill/Plugin 污染

**定义**：由于加载了过多或冲突的 Skills/Plugins 导致的上下文污染。

**检测指标**：
- 功能重叠度 > 80%
- 权限冲突
- 命名空间冲突

**修复策略**：
- 禁用冗余 Skills
- 合并功能重叠的 Skills
- 规范化命名

### Type 2: 指令矛盾

**定义**：用户在对话的不同阶段提出了互相矛盾的指令。

**检测指标**：
- 目标向量相反（余弦相似度 < -0.5）
- 约束条件互斥
- 偏好在短时间内大幅变更

**修复策略**：
- 生成澄清声明
- 建议重置特定段落
- 整理有效约束

### Type 3: 错误累积

**定义**：早期的一个小错误导致后续所有推理都基于错误假设。

**检测指标**：
- 推理链中存在错误节点
- 后续推理依赖错误节点
- 误解持续超过 N 轮对话

**修复策略**：
- 识别最佳回溯点
- 生成纠正声明
- 提供验证清单

## 框架适配层

### Claude Code Adapter

**配置位置**：
- 全局：`~/.claude/skills/`
- 项目：`.claude/skills/`

**实现方式**：
- Skill 系统：YAML frontmatter + Markdown 内容
- 自动触发：基于 description 匹配
- 手动触发：`/contextdoctor` 或 `/repair`

### Codex CLI Adapter

**配置位置**：
- 全局：`~/.codex/codex.md`
- 项目：`./codex.md`
- Agent：`./agents/context-doctor/instructions.md`

**实现方式**：
- 指令定义：Markdown 格式
- Agent 专用：可创建独立 Agent
- 手动触发：`/contextdoctor` 或 `/repair`

### Cursor Adapter

**配置位置**：
- 全局：`~/.cursor/commands/`
- 项目：`.cursor/commands/`
- 规则：`.cursorrules` 或 `.cursor/rules/`

**实现方式**：
- Custom Commands：JSON 格式定义
- Project Rules：Markdown 规则文件
- Notepads：可引用的笔记本

### OpenCode/Crush Adapter

**配置位置**：
- 全局：`~/.config/opencode/commands/`
- 项目：`.opencode/commands/` 或 `.crush/`

**实现方式**：
- Markdown 命令：YAML frontmatter + 模板
- JSON 配置：`opencode.jsonc` 或 `.crush.json`
- 插件系统：JavaScript 模块

## 报告设计系统

基于 Starbucks 设计系统，使用暖色调营造舒适的阅读体验。

### 色彩系统

| 颜色 | Hex | 用途 |
|------|-----|------|
| Starbucks Green | #006241 | 品牌标识、标题 |
| Green Accent | #00754A | CTA 按钮、强调 |
| House Green | #1E3932 | 深色背景、底部 |
| Neutral Warm | #f2f0eb | 页面背景 |
| Gold | #cba258 | 警告级别 |
| Red | #c82014 | 严重级别 |

### 组件规范

- **卡片**：12px 圆角，柔和阴影
- **按钮**：50px 全圆角
- **评分圆环**：SVG 实现，动态动画
- **图表**：CSS 条形图，响应式高度

## 扩展性设计

### 添加新的检测规则

1. 在 `detection/` 目录创建新的检测器
2. 实现 `detect()` 方法，返回问题列表
3. 在核心引擎中注册
4. 更新配置文件的 severity 设置

### 添加新的框架支持

1. 在 `plugins/` 创建新的适配器目录
2. 实现框架特定的配置格式
3. 添加安装脚本支持
4. 更新文档

### 添加新的语言支持

1. 在 `i18n/` 添加新的语言文件
2. 翻译所有 UI 文本
3. 更新 `supportedLanguages` 配置
4. 添加 README 语言版本

## 性能考虑

- **增量检测**：只分析新增加的对话内容
- **缓存机制**：缓存 Skills 元数据
- **异步处理**：报告生成使用异步
- **上下文限制**：遵守各框架的上下文限制

## 安全考虑

- 不发送用户代码到外部服务
- 所有分析在本地完成
- 报告文件保存在用户指定位置
- 可选的敏感信息脱敏
