# 要解决的问题/痛点
1. 问题一：用户在使用Agent产品/cli的时候，常常忽略自己的提示词中前后矛盾的地方，导致上下文污染
2. 问题二: skill太多，plugin太多，使用时会相互冲突，上下文污染，还导致加载过慢的问题，用户也很难肉眼修复

# 适配架构
opencode
codex
claude code
cursor
openclaw

# 使用方式
用户直接用指令驱动
/contextdoctor 检查当前上下文的污染，输出html可视化报告
/repair 不仅检查当前上下文的污染，输出html可视化报告，还提供解决方案

# 特性
极简安装，快速上手
易于改装，框架灵活
Readme多语言支持，打造GitHub仓库品牌，增强宣传推广效果。

# 污染类型
1. skill, plugin污染，互相冲突，或冗余
2. 需求冲突，前后用户指令不一致
3. 某个位置的错误，可能导致错误的累积

# 可视化报告
1. 支持多语言
2. 严重等级划分，颜色区分
3. 消息可视化
4. 设计风格参考DESIGN.md

# 具体实现方式
1. 用户运行对应指令的时候，将指令直接转化为对于当前Agent的指令（让Agent反思自己当前指令中是否有上下文污染的情况，给出综合评分，按照给定的报告格式，输出可视化报告）
2. 安装的时候，直接将指令全局安装，用户在任何位置使用Agent框架，都可以使用上下文检查指令。
3. 指令的实现方式需要参考每个具体的 Agent框架 里面定制指令的方式（见COMMANDS_REFERENCE.md）。
